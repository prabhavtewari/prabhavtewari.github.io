Interaction System

The Grab Field

After a few iterations, the interaction system landed on a grab field. It deviates slightly from the pool theme, but it is a fair tradeoff given how the alternatives performed in practice. The field is tied to a predefined attach point on the cue prefab and is visualised as a set of sci-fi style concentric rings hovering over the tip of the cue.

Grab Field Visual

The player holds B to activate grab mode. Once active, a trigger event fires when the attach point overlaps with any object. The colliding object is checked against the interaction task target, and if it matches, it is reparented to the cue at the attach point transform. Tapping B again releases the object, removing it from the cue's transform hierarchy and applying a short cooldown to prevent it from being immediately picked up again.

To avoid unintended attachments, the trigger is filtered by direction. The object only attaches if it lies in front of the attach point at the moment of contact, checked using a dot product between the attach point's forward vector and the direction to the object. A negative or zero result is ignored.

Translation

Since the attached object is reparented to the cue, translation comes for free. The cue already moves through 3D space driven by the two-handed grip system, so the object follows without any additional logic needed.

Rotation

Rotation is split across three axes, each handled differently. The first axis is the roll of the cue itself. As shown in UpdateStickPosition, the cue's roll is driven by the striking hand's up direction, projected onto the plane perpendicular to the stick. Rolling the controller therefore rolls the cue, and because the object is parented to it, the object rolls with it.

The remaining two axes are mapped directly to the left joystick. Left and right on the joystick rotate the object in yaw, while up and down control pitch. Roll is excluded here, as it is already determined by the striking hand's up direction projected onto the plane perpendicular to the stick. A deadzone is applied along with an axis priority threshold, so that only the dominant axis drives rotation at any given moment, preventing diagonal drift and keeping rotations deliberate.
